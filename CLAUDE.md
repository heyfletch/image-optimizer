# Image Optimizer — Claude Code Notes

## What this project is

A macOS image optimization tool with two interfaces:
1. **Tauri desktop app** — React frontend + Rust shell + Node.js sidecar (Sharp/SVGO)
2. **Finder integration** — shell scripts invoked as Finder toolbar buttons or Quick Actions

## Architecture

```
src/           React UI (Vite + TypeScript)
src-tauri/     Rust/Tauri shell — spawns sidecar, manages window
sidecar/       Node.js image processing (Sharp for raster, SVGO for SVG)
               Communicates via JSON over stdin/stdout (IPC mode) or CLI flags
finder-actions/  macOS Finder integration
  *.sh                   Action scripts (called per-file, args = file paths)
  toolbar-apps/          .app bundles dragged into Finder toolbar
    */Contents/MacOS/run   Gets Finder selection via AppleScript, calls the .sh
  create-toolbar-apps.sh   Regenerates toolbar-apps/ from templates
  install.sh               Installs Automator Quick Actions to ~/Library/Services/
```

## Finder integration flow

Toolbar .app `run` script → AppleScript gets selected Finder files → calls `finder-actions/<action>.sh "$file"` per file → sidecar processes → done.

Quick Actions: Automator workflows in `~/Library/Services/` call the same `.sh` scripts directly with selected files as arguments.

## Key dev commands

```bash
npm run tauri dev          # dev mode (hot-reloads frontend)
cd sidecar && npx tsc      # rebuild sidecar after TS changes
cd sidecar && npx vitest   # run sidecar tests
npm run tauri build        # production .app bundle
cd finder-actions && ./create-toolbar-apps.sh  # rebuild toolbar .app bundles
cd finder-actions && ./install.sh              # install Quick Actions
```

## Notes

- Toolbar app `run` scripts are generated by `create-toolbar-apps.sh` — if you edit individual `run` files, also update the template in that script (line ~47) so they stay in sync on next regeneration.
- Node.js path is resolved via nvm at runtime; no bundled Node.js.
- Notifications: intentionally removed. The scripts previously used `osascript display notification` which bypasses macOS notification settings. Operations now run silently.
